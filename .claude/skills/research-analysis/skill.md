# 연구 분석

## 사용 시점

- 새로운 라이브러리/프레임워크 평가
- 아키텍처 설계 의사결정 (패턴, 기술 선택)
- 성능 병목 원인 조사
- 보안 이슈 분석 및 해결책 수립

## 절차

1. 조사 질문 정의

   - 핵심 질문 명확화
   - 평가 기준 수립 (성능, 보안, 유지보수성, 비용)
   - 기존 제약조건 확인
2. 현황 분석 (코드베이스)

   - 기존 관련 코드/아키텍처 검토
   - 현재 병목/문제점 식별
   - 스택 내 이미 사용 중인 유사 기술 확인
3. 옵션 평가

   - 각 옵션의 장단점 작성
   - 프로토타입/PoC 코드 샘플 제시 (필요시)
   - 기술 성숙도/커뮤니티 지원 확인
   - 점수 평가 (기준별 1-10점)
4. 권장사항 및 마이그레이션 계획 (해당 시)

   - 최적 선택 정당화
   - 단계별 적용 계획
   - 위험 요소 및 완화책
   - 테스트 전략
5. 문서화

   - 의사결정 기록 (ADR: Architecture Decision Record)
   - 설명 요약 (팀이 이해할 수 있는 수준)
   - 향후 검토 시점 제안
