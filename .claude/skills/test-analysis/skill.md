# Test Analysis

## 사용 시점
- 새 기능의 테스트 계획 수립
- 버그 재현 테스트 케이스 작성
- 기존 코드의 테스트 커버리지 분석
- 테스트 실패 원인 분석

## 절차

1. 테스트 대상 이해
   - SPEC.md 또는 구현 코드 읽기
   - 테스트 가능한 단위 식별
   - Happy path vs error path 분류

2. 테스트 계획 수립
   - Unit test 케이스 (각 함수/클래스)
   - Integration test 케이스 (모듈 간 상호작용)
   - Edge case 식별 (경계값, 극한 상황)
   - Security test 케이스 (입력 검증, 권한 검사)

3. 테스트 케이스 작성
   - Arrange-Act-Assert 패턴으로 코드화
   - Mock 외부 의존성 (API, 파일 시스템)
   - 각 케이스에 명확한 설명 주석

4. 실패 케이스 분석
   - 실패한 테스트 읽기
   - 예상값 vs 실제값 비교
   - 근본 원인 파악 (타입, 로직, 의존성)
   - 수정 제안 (코드 수정 또는 테스트 수정)

5. 커버리지 검증
   - 전체 함수/라인 커버리지 계산
   - 누락된 테스트 식별
   - 커버리지 리포트 생성
