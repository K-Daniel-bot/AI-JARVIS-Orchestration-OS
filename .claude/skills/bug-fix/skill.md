# 버그 수정

## 사용 시점

- 테스트 실패 원인 수정
- 런타임 에러/예외 해결
- 의도치 않은 동작 수정
- 엣지 케이스 버그 처리

## 절차

1. 버그 재현

   - 버그 리포트 또는 실패한 테스트 읽기
   - 재현 조건 정확히 파악 (입력값, 환경, 순서)
   - 재현 가능한 최소 케이스 작성
   - 에러 메시지/스택 트레이스 전체 기록
2. 근본 원인 분석

   - 에러 발생 위치 (파일:라인) 정확히 식별
   - 원인 분류:
     - Type error: 타입 불일치 또는 null/undefined
     - Logic error: 조건/계산 오류
     - Async error: 레이스 컨디션 또는 Promise 미처리
     - Dependency error: 패키지 버전 충돌 또는 missing import
     - State error: 잘못된 상태 전이 또는 XState 오류
   - 원인 파악 전 코드 수정 금지
3. 수정 계획

   - 최소 변경 원칙 (가장 작은 수정으로 해결)
   - 부작용 범위 예측 (수정이 영향 미치는 코드)
   - 수정 방법 2가지 비교 후 최적 선택
4. 수정 실행

   - 최소한의 코드만 변경
   - `any` 타입 도입 금지
   - 수정 후 `pnpm typecheck` 실행
5. 검증

   - 회귀 테스트 작성 (이 버그가 다시 발생 시 탐지)
   - `pnpm test` 전체 실행 (새 실패 없음 확인)
   - 원본 재현 케이스 통과 확인
   - /bug-fix-review 스킬로 수정 검토
6. 문서화

   - 버그 원인 1-2문장 요약
   - 수정 내용 설명
   - 예방 방법 제안 (향후 유사 버그 방지)
