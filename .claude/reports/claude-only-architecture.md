### 3.2 비용 관점

**번들 효과 (Bundle Discount)**

Anthropic의 **Batch API**를 사용하면:
```
표준 가격 (실시간 API):
- Opus: $5 input / $25 output
- Sonnet: $3 input / $15 output
- Haiku: $1 input / $5 output

Batch API 가격 (50% 할인):
- Opus: $2.50 input / $12.50 output  ← 50% 절감
- Sonnet: $1.50 input / $7.50 output  ← 50% 절감
- Haiku: $0.50 input / $2.50 output   ← 50% 절감

결과: 전체 에이전트 실행 시 50% 비용 절감 가능
```

**Prompt Caching** (매우 중요)

```
시나리오: Spec Agent, Planner, Codegen, Review가 모두
         같은 사용자 코드베이스(10MB)를 분석

표준: 각 에이전트가 전체 코드베이스 입력
      10MB × 4 에이전트 = 40MB 입력 비용

Prompt Caching:
1. 코드베이스를 캐시 가능 프롬프트 토큰으로 변환 (1회)
2. 첫 에이전트가 읽을 때만 전체 비용 청구
3. 이후 에이전트는 캐시된 토큰만 90% 할인된 가격으로 읽음
   = 40MB 입력 중 90%를 할인가로 처리
   = 대략 83% 비용 절감

실제 효과: 동일 코드베이스 N개 에이전트 분석 시
          1회 + (N-1)회 × 10% = 1 + 0.1(N-1) 배의 비용만 필요
```

```python

@dataclass
class AgentTask:
    agent_type: str  # "Orchestrator" | "Codegen" | "Review"
    priority: str    # "realtime" | "batch"

if task.priority == "batch":
    # Batch API (50% 할인)
    # 24시간 이내에 처리됨을 감수
    costs = costs * 0.5
else:
    # 실시간 API (표준 가격)
    pass
```

**예상 효과:**
```
월 실행 1000회 기준:
- 실시간 필요: 10% (100회) = 100 × $5 (평균) = $500
- 배치 가능: 90% (900회) = 900 × $2.5 (할인) = $2,250
────────────────────────────────────────────
총 월 비용: $2,750 (vs 단순 실시간 $5,000 = 45% 절감)
```
